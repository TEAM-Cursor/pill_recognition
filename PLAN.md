## 목표 / 범위

알약을 카메라로 비추면, 미리 입력해 둔 **개인 정보(나이·성별·임신/수유·복용약·알레르기)** 와 **현재 시각**에 맞춰 LLM이 용법·용량·주의사항을 텍스트로 안내하는 모바일 앱. (예: "지금 21시라 공복 복용 시 다음날 아침 속쓰림이 있을 수 있어요.")

**범위 (포함)**
- 온보딩: 권한 요청 → 개인정보 입력 (최초 1회).
- 홈 = 카메라 즉시 활성화. 중앙 정사각 뷰파인더에 알약을 맞춰 정지하면 자동 인식(별도 촬영 버튼 없음) → "인식 완료" 알림.
- 결과 = LLM 텍스트 안내. **답변마다 "요약" 버튼** → 대화 내용을 문서로 요약, 질문이 쌓일 때마다 **새 버전(v1→v2→…)** 누적 생성.
- 하단 3탭, **좌우 스와이프**: (왼) 내 알약사전 = 대화 기록 열람·이어쓰기 / (중앙) 홈 = 카메라 / (오른) 기타 = 카드 2개(내 정보 입력, 증상별 약 추천).

**현재 단계**: 더미 데이터로 **UI 프로토타입 먼저**. 실제 공공 API·LLM 연동은 다음 단계.

**범위 밖**: 실제 의료 진단·처방, 결제, 약 구매.

## 기술 결정

- **플랫폼**: React Native (Expo, TypeScript, expo-router). 카메라+권한+크로스플랫폼, Windows 개발 + 실폰 QR 테스트가 쉬움.
- **LLM / 비전**: Claude (Anthropic) — 비전+텍스트 단일 모델로 사진 분석과 안내·요약을 함께.
- **알약 인식 파이프라인**: 의약품 낱알식별 정보 API는 **이미지를 직접 받지 못하고** 색/모양/각인(표시문자)/분할선 등 **텍스트 속성으로 검색**한다. 따라서 → ① 비전 LLM이 사진에서 각인·색·모양·분할선을 추출 → ② 낱알식별 API로 품목 매칭 → ③ 매칭된 품목코드로 DUR·e약은요 조회.
- **공공데이터포털 3종**(인증키 필요, 현재 더미):
  1. 의약품 낱알식별 정보
  2. 의약품안전사용서비스(DUR) 품목정보
  3. 의약품개요정보(e약은요)
- **네비게이션**: SDK 56부터 expo-router가 react-navigation과 비호환 → `expo-router/js-top-tabs`의 `TopTabs`(머티리얼 탑탭)를 `tabBarPosition: bottom`으로 배치해 좌우 스와이프 애니메이션 구현.
- **개인 건강정보 저장**: 단말 로컬(AsyncStorage / SecureStore). 서버 미전송.
- **UI 디자인**: `C:\dev\docs\web-design` 프레임워크 적용. 확정 토큰·패턴은 `pill_recognition/.design/system.md`에 축적.

**UI 기본값 (확정, 변경 가능)**
- 인식 성공 대화는 알약사전에 자동 저장 · 요약 버튼은 LLM 답변마다(내 질문엔 없음) · 요약 문서는 누적형(v3 = v1+v2+신규) · 증상별 약 추천은 OTC(일반의약품) 한정 + 면책 문구 · 권한 거부 시 안내 화면 후 재요청.
